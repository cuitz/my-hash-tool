# 哈希计算器设计文档

> 当前版本：v1.3.1

## 1. 插件目标

开发一个 ZTools 哈希计算插件，用于快速计算本地文件或字符串的哈希值。

默认支持：

- MD5
- SHA1
- SHA256

可选额外算法（用户在设置中开启）：

- CRC32
- SHA384
- SHA512

插件重点是简单、稳定、低内存占用。文件场景适合用户对下载文件、安装包、压缩包、镜像文件等进行基础完整性校验；字符串场景适合快速计算文本、Token、配置片段等内容的哈希值。

## 2. 功能范围

### 包含功能

- 在同一个输入框内输入或粘贴字符串计算 Hash
- 在同一个输入区域内拖入文件计算 Hash
- 点击选择文件计算 Hash
- 支持单文件和多文件
- 每个任务显示独立计算进度
- 每个任务显示已启用算法的结果
- 支持复制单个 Hash 值
- 支持复制单个任务全部 Hash
- 支持复制全部任务结果
- 大文件计算前提醒
- 用户可在设置页开启/关闭大文件提醒
- 用户可自定义大文件提醒阈值，单位为 GB
- 使用流式读取，避免大文件占用过多内存
- 可选开启 CRC32、SHA384、SHA512 额外算法
- Hash 结果支持大小写切换（逐行或批量）
- 设置默认大小写，影响后续新计算结果的初始显示
- 复制遵循所见即所复制，按当前显示大小写输出
- 批量任务完成后显示轻提示
- 支持取消批量计算中的剩余任务

### 不包含功能

- 不做 Hash 对比
- 不做批量导出
- 不做历史记录
- 不做文件夹递归
- 不做并发计算

## 3. 信息架构

插件包含两个主要页面：

- 主页面：统一输入、文件拖拽、文件选择、进度、结果展示
- 设置页面：大文件提醒、默认大小写、额外算法开关

### 4.2 顶部区域

顶部左侧显示当前已启用的算法列表（如 `MD5 / SHA1 / SHA256`）。

顶部右侧按需显示操作按钮：

- 计算中：显示「取消」按钮
- 有任务且未计算中：显示「清空」按钮
- 有已完成任务：显示「全部小写」「全部大写」「复制全部结果」按钮
- 始终显示「设置」按钮

### 4.3 统一输入区域

统一输入区域支持三种方式：

- 输入或粘贴字符串
- 拖入文件
- 点击选择文件

默认状态：

```text
输入字符串或拖入文件
在这里输入或粘贴字符串；也可以把文件拖到这个输入框区域。
```

拖拽悬停状态：

```text
松开开始计算文件 Hash
```

输入字符串后，点击「计算字符串」或按 ⌘/Ctrl+Enter 开始计算。

选择或拖入文件后，插件先读取文件大小，再根据设置判断是否显示大文件提醒。

### 4.4 批量进度面板

选择多个文件后，在输入区域下方显示批量进度面板：

```text
批量计算中  64%
████████████░░░░░░░░  3 项  1 完成  1 计算中  1 等待
```

状态标签包括：总项数、完成数、计算中数、等待数、失败数（有失败时才显示）。

### 4.5 结果卡片

每个文件或字符串任务使用独立结果卡片。

卡片内容：

- 任务名称（文件名或「字符串输入」）
- 文件路径或字符串字符数/字节数
- 文件大小或「UTF-8」标识
- 计算完成时显示「复制全部」按钮
- 当前状态文本
- 当前进度百分比
- 进度条
- 每个已启用算法的 Hash 行：算法名、Hash 值、大小写切换按钮、复制按钮

## 5. 大小写切换设计

### 5.1 逐行切换

每个 Hash 行右侧提供独立切换按钮：

- 当前为小写时显示 `aa`，点击切换为大写
- 当前为大写时显示 `Aa`，点击切换为小写
- 计算未完成时按钮保留焦点可达性，但不触发切换

### 5.2 批量切换

顶部操作区提供「全部小写」「全部大写」按钮，一次性将所有已完成任务的所有算法统一切换。

### 5.3 切换行为

- 切换为纯渲染层操作，不触发任何重新计算
- 复制遵循所见即所复制：单值复制、卡片复制全部、复制全部结果均按当前显示大小写输出

### 6.2 设置项

当前包含四个设置项：

```json
{
  "largeFileWarningEnabled": true,
  "largeFileWarningThresholdGB": 2,
  "extraAlgorithms": [],
  "defaultCase": "lowercase"
}
```

说明：

- `largeFileWarningEnabled`：是否开启大文件提醒，默认 `true`
- `largeFileWarningThresholdGB`：提醒阈值，单位 GB，默认 `2`，范围 1–1024，保留两位小数
- `extraAlgorithms`：已启用的额外算法列表，默认空数组，可选值 `crc32`、`sha384`、`sha512`
- `defaultCase`：新计算结果的默认大小写，默认 `lowercase`，可选 `uppercase`

### 6.3 阈值输入规则

- 单位：GB
- 默认值：2
- 最小值：1
- 最大值：1024
- 输入无效时恢复默认值 2
- 步进为 1GB

## 7. 大文件提醒设计

### 7.1 触发条件

如果开启了大文件提醒，满足任一条件时显示提醒：

- 单个文件大小超过用户设置的阈值
- 本次选择的文件总大小超过用户设置的阈值

### 7.2 提醒内容

```text
大文件提醒
计算可能需要较长时间

本次选择 3 个文件，共 4.8 GB。插件会使用流式读取，不会一次性加载整个文件。

[ ] 不再提醒大文件计算

[取消] [继续计算]
```

### 7.3 用户行为

用户可以选择：

- 继续计算
- 取消
- 勾选「不再提醒大文件计算」后继续计算，此时更新设置 `largeFileWarningEnabled: false`

## 8. 计算策略

### 8.1 字符串计算

字符串使用 UTF-8 编码，同时计算所有已启用算法。

```js
crypto.createHash('sha256').update(text, 'utf8').digest('hex')
```

### 8.2 文件流式读取

大文件使用 Node.js 流式读取，不允许一次性读取完整文件。

```js
fs.createReadStream(filePath, {
  highWaterMark: 4 * 1024 * 1024
})
```

### 8.3 单次读取计算所有算法

每个文件只读取一次，同时计算所有已启用算法：

```text
文件流
  ├─ MD5
  ├─ SHA1
  ├─ SHA256
  ├─ CRC32（如已启用）
  ├─ SHA384（如已启用）
  └─ SHA512（如已启用）
```

### 8.4 多文件串行计算

多文件采用串行计算：

```text
file 1 完成
  ↓
file 2 完成
  ↓
file 3 完成
```

原因：降低内存波动、降低磁盘压力、简化进度展示、避免多个大文件同时读取导致性能下降。

## 9. 进度设计

每个任务单独维护进度。

字符串任务通常会立即完成，UI 会短暂显示计算中状态（约 80ms）再进入完成。

文件任务进度计算：

```text
progress = 已读取字节数 / 文件总字节数
```

状态变化：

```text
waiting -> hashing -> done
waiting -> hashing -> error
```

任务数据结构：

```ts
type HashAlgorithm = 'md5' | 'sha1' | 'sha256' | 'crc32' | 'sha384' | 'sha512'
type CoreHashAlgorithm = 'md5' | 'sha1' | 'sha256'
type ExtraHashAlgorithm = 'crc32' | 'sha384' | 'sha512'
type HashCase = 'lowercase' | 'uppercase'
type TaskStatus = 'waiting' | 'hashing' | 'done' | 'error'

type BaseHashTask = {
  id: string
  kind: 'file' | 'text'
  name: string
  size: number
  status: TaskStatus
  progress: number
  error: string
  algorithms: HashAlgorithm[]
  hashes: Record<HashAlgorithm, string> | null
  displayCases: Partial<Record<HashAlgorithm, HashCase>>
}

type FileHashTask = BaseHashTask & {
  kind: 'file'
  path: string
}

type TextHashTask = BaseHashTask & {
  kind: 'text'
  path: ''
  text: string
}
```

## 11. 视觉风格

整体风格：安静、可靠、工具感。

设计原则：

- 不做营销页
- 不使用大面积装饰
- 结果优先，字符串和文件共用同一个输入区域
- Hash 使用等宽字体
- 进度条放在每个任务卡片中
- 复制按钮保持轻量
- 错误状态清晰但不刺眼

建议颜色：

- 背景：浅灰白
- 主文本：深灰黑
- 次级文本：中灰
- 进度条：蓝绿色或绿色
- 错误状态：低饱和红色
- 边框：浅灰

## 12. 复制设计

复制层级：

- 单值复制：点击 Hash 值本身或右侧复制按钮，复制该算法当前显示的 Hash 字符串
- 卡片复制全部：点击卡片右上角「复制全部」，复制该任务所有算法的结果，格式为文件路径 + 各算法行
- 全局复制全部结果：点击顶部「复制全部结果」，复制所有已完成任务，任务间以空行分隔

复制成功后按钮状态短暂变化（约 1.5 秒后恢复）：

```text
[复制] -> [已复制]
[复制全部] -> [已复制]
[复制全部结果] -> [已复制全部]
```

复制内容遵循所见即所复制，按当前显示大小写输出。

## 13. 错误处理

需要处理的错误：

- 文件不存在
- 没有读取权限
- 文件读取中断
- 文件大小读取失败
- Hash 计算异常

错误展示在对应文件卡片内，不影响其他文件继续计算。

取消批量计算时，已完成的任务结果保留，剩余任务标记为「已取消」。

## 14. 存储设计

设置优先使用 ZTools 本地存储（`window.ztools.dbStorage`）。

```json
{
  "largeFileWarningEnabled": true,
  "largeFileWarningThresholdGB": 2,
  "extraAlgorithms": [],
  "defaultCase": "lowercase"
}
```

如果 ZTools 存储不可用，降级使用 `localStorage`。读取时做完整的字段校验和范围检查，损坏时回退到默认值。
